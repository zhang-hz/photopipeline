## 二、功能设计（Functional Design）

### 2.1 职责边界

MainWindow 是整个应用的**外壳容器**，负责：
- 承载 5 个子视图（Filmstrip / PluginBrowser / Preview / PipelineEditor / Batch）
- 管理后端连接状态指示
- 提供全局状态消息通道（SnackbarPresenter）
- 提供窗口级快捷键
- 提供主题切换入口

### 2.2 数据流向

```
MainViewModel
├── Filmstrip ───────────→ 选择图像 ──→ PreviewViewModel (加载预览)
├── PluginBrowser ───────→ 拖放插件 ──→ PipelineEditorViewModel (添加节点)
├── Preview ─────────────→ (独立渲染，不与其他 VM 直接通信)
├── PipelineEditor ──────→ 执行管线 ──→ PreviewViewModel (更新 AfterBitmap)
│                        → 验证状态 ──→ MainWindow (标题栏显示 Valid: ✓/✗)
├── Batch ───────────────→ (独立工作流)
└── IsBackendHealthy ───→ MainWindow (状态栏指示器)
```

### 2.3 用户操作流程

| 用户操作 | MainWindow 响应 | 子视图响应 |
|---------|---------------|-----------|
| Ctrl+=/-/0 | 转发到 Preview.ZoomIn/Out/Reset | Preview 缩放 |
| 点击 Theme 按钮 | OnThemeToggleClick → 切换 Dark/Light | 全局主题变更 |
| 点击 Settings 按钮 | ShowSettingsCommand → 打开 SettingsDialog | 设置对话框 |
| 调整分隔线 | GridSplitter 拖拽 | 各面板宽度实时变化 |
| 窗口关闭 | Closing → ViewModel.Shutdown() | 各子 VM 清理 |

### 2.4 与其他组件的协作

| 协作方 | 关系 | 实现方式 |
|-------|------|---------|
| SettingsViewModel | 主题变更由 MainWindow 直接处理（代码后置），不通过 VM | 事件处理器 |
| 各子 VM | MainViewModel 订阅子 VM 的 StatusMessage 变更并转发 | PropertyChanged 事件 |
| BackendService | MainViewModel 定期轮询后端健康状态 | 定时器 → IsBackendHealthy |

### 3.3 布局偏离分析

设计文档的布局与实际实现的布局有 **显著差异**：

| 区域 | 设计文档位置 | 实际位置 | 影响 |
|------|-----------|---------|------|
| 插件浏览器 | 右栏（插件控制面板） | 左栏下方 | 用户预期在右侧找到插件配置，实际在左侧 |
| 批量处理 | 左栏下方 | 右栏下方 | 批量与管线编辑同侧，更符合工作流直觉 |
| 管线总览（迷你 DAG） | 中栏下方预览区下方 | 右侧全高度 | 实现更合理——DAG 编辑器需要更多空间 |
| 信息栏 | 中栏底部 | 底部状态栏 | 合并为全局状态栏，覆盖信息栏功能 |

**结论：** 实际布局在设计上更合理（管线编辑器需要最大垂直空间），但偏离了原始设计文档。建议更新设计文档反映实际布局。

### 3.4 交互缺失清单

1. **无窗口位置保存/恢复** — 重启后窗口回到居中位置，丢失用户上次的位置偏好
2. **无文件拖入支持** — 用户不能从资源管理器拖放图像到窗口
3. **主题切换无法通过快捷键** — 仅通过标题栏按钮点击
4. **后端断连无主动重连按钮** — 状态栏显示断连但无"重试"操作
